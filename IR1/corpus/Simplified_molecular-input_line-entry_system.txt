<h1 class="firstHeading" id="firstHeading" lang="en">Simplified molecular-input line-entry system</h1> <div class="mw-body-content" id="bodyContent">
<div class="noprint" id="siteSub">From Wikipedia, the free encyclopedia</div> <div id="contentSub"></div>

<p>The <b>simplified molecular-input line-entry system</b> (<b>SMILES</b>) is a specification in form of a <a href="/wiki/Line_notation" title="Line notation">line notation</a> for describing the structure of <a href="/wiki/Chemical_species" title="Chemical species">chemical species</a> using short <a href="/wiki/ASCII" title="ASCII">ASCII</a> <a href="/wiki/String_(computer_science)" title="String (computer science)">strings</a>. SMILES strings can be imported by most <a href="/wiki/Molecule_editor" title="Molecule editor">molecule editors</a> for conversion back into <a class="mw-redirect" href="/wiki/Two-dimensional" title="Two-dimensional">two-dimensional</a> drawings or <a href="/wiki/Dimension" title="Dimension">three-dimensional</a> models of the molecules.
</p><p>The original SMILES specification was initiated in the 1980s. It has since been modified and extended. In 2007, an <a href="/wiki/Open_standard" title="Open standard">open standard</a> called OpenSMILES was developed in the open-source chemistry community.  Other linear notations include the <a href="/wiki/Wiswesser_line_notation" title="Wiswesser line notation">Wiswesser line notation</a> (WLN), <a class="new" href="/w/index.php?title=ROSDAL&amp;action=edit&amp;redlink=1" title="ROSDAL (page does not exist)">ROSDAL</a>, and <a class="mw-redirect" href="/wiki/SYBYL_Line_Notation" title="SYBYL Line Notation">SYBYL Line Notation</a> (SLN).
</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The original SMILES specification was initiated by David Weininger at the USEPA Mid-Continent Ecology Division Laboratory in <a href="/wiki/Duluth,_Minnesota" title="Duluth, Minnesota">Duluth</a> in the 1980s.<sup class="reference" id="cite_ref-SMILES1_1-0"><a href="#cite_note-SMILES1-1">[1]</a></sup><sup class="reference" id="cite_ref-SMILES2_2-0"><a href="#cite_note-SMILES2-2">[2]</a></sup><sup class="reference" id="cite_ref-SMILES3_3-0"><a href="#cite_note-SMILES3-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> Acknowledged for their parts in the early development were "Gilman Veith and Rose Russo (USEPA) and Albert Leo and <a href="/wiki/Corwin_Hansch" title="Corwin Hansch">Corwin Hansch</a> (Pomona College) for supporting the work, and Arthur Weininger (Pomona; Daylight CIS) and Jeremy Scofield (Cedar River Software, Renton, WA) for assistance in programming the system."<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> The <a class="mw-redirect" href="/wiki/Environmental_Protection_Agency" title="Environmental Protection Agency">Environmental Protection Agency</a> funded the initial project to develop SMILES.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup><sup class="reference" id="cite_ref-SMILES_Tutorial:_What_is_SMILES?_7-0"><a href="#cite_note-SMILES_Tutorial:_What_is_SMILES?-7">[7]</a></sup>
</p><p>It has since been modified and extended by others, most notably by <a class="new" href="/w/index.php?title=Daylight_Chemical_Information_Systems&amp;action=edit&amp;redlink=1" title="Daylight Chemical Information Systems (page does not exist)">Daylight Chemical Information Systems</a>. In 2007, an <a href="/wiki/Open_standard" title="Open standard">open standard</a> called "OpenSMILES" was developed by the <a href="/wiki/Blue_Obelisk" title="Blue Obelisk">Blue Obelisk</a> open-source chemistry community.  Other 'linear' notations include the <a class="mw-redirect" href="/wiki/Wiswesser_Line_Notation" title="Wiswesser Line Notation">Wiswesser Line Notation</a> (WLN), <a class="new" href="/w/index.php?title=ROSDAL&amp;action=edit&amp;redlink=1" title="ROSDAL (page does not exist)">ROSDAL</a> and <a class="mw-redirect" href="/wiki/SYBYL_Line_Notation" title="SYBYL Line Notation">SLN</a> (Tripos Inc).
</p><p>In July 2006, the <a href="/wiki/International_Union_of_Pure_and_Applied_Chemistry" title="International Union of Pure and Applied Chemistry">IUPAC</a> introduced the <a href="/wiki/International_Chemical_Identifier" title="International Chemical Identifier">InChI</a> as a standard for formula representation.  SMILES is generally considered to have the advantage of being slightly more human-readable than InChI; it also has a wide base of software support with extensive theoretical (e.g., <a href="/wiki/Graph_theory" title="Graph theory">graph theory</a>) backing.
</p>
<h2><span class="mw-headline" id="Terminology">Terminology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=2" title="Edit section: Terminology">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The term SMILES refers to a line notation for encoding molecular structures and specific instances should strictly be called SMILES strings. However, the term SMILES is also commonly used to refer to both a single SMILES string and a number of SMILES strings; the exact meaning is usually apparent from the context. The terms "canonical" and "isomeric" can lead to some confusion when applied to SMILES.  The terms describe different attributes of SMILES strings and are not mutually exclusive.
</p><p>Typically, a number of equally valid SMILES strings can be written for a molecule.  For example, <code>CCO</code>, <code>OCC</code> and <code>C(O)C</code> all specify the structure of <a href="/wiki/Ethanol" title="Ethanol">ethanol</a>. Algorithms have been developed to generate the same SMILES string for a given molecule; of the many possible strings, these algorithms choose only one of them.  This SMILES is unique for each structure, although dependent on the <a href="/wiki/Canonicalization" title="Canonicalization">canonicalization</a> algorithm used to generate it, and is termed the canonical SMILES.  These algorithms first convert the SMILES to an internal representation of the molecular structure; an algorithm then examines that structure and produces a unique SMILES string.  Various algorithms for generating canonical SMILES have been developed and include those by <a class="new" href="/w/index.php?title=Daylight_Chemical_Information_Systems&amp;action=edit&amp;redlink=1" title="Daylight Chemical Information Systems (page does not exist)">Daylight Chemical Information Systems</a>, <a href="/wiki/OpenEye_Scientific_Software" title="OpenEye Scientific Software">OpenEye Scientific Software</a>, <a class="new" href="/w/index.php?title=MEDIT&amp;action=edit&amp;redlink=1" title="MEDIT (page does not exist)">MEDIT</a>, <a href="/wiki/Chemical_Computing_Group" title="Chemical Computing Group">Chemical Computing Group</a>, <a class="new" href="/w/index.php?title=MolSoft_LLC&amp;action=edit&amp;redlink=1" title="MolSoft LLC (page does not exist)">MolSoft LLC</a>, and the <a href="/wiki/Chemistry_Development_Kit" title="Chemistry Development Kit">Chemistry Development Kit</a>. A common application of canonical SMILES is indexing and ensuring uniqueness of molecules in a <a href="/wiki/Chemical_database" title="Chemical database">database</a>.
</p><p>The original paper that described the CANGEN<sup class="reference" id="cite_ref-SMILES2_2-1"><a href="#cite_note-SMILES2-2">[2]</a></sup> algorithm claimed to generate unique SMILES strings for graphs representing molecules, but the algorithm fails for a number of simple cases (e.g. <a href="/wiki/Cuneane" title="Cuneane">cuneane</a>, 1,2-dicyclopropylethane) and cannot be considered a correct method for representing a graph canonically.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup>  There is currently no systematic comparison across commercial software to test if such flaws exist in those packages.
</p><p>SMILES notation allows the specification of <a href="/wiki/Molecular_configuration" title="Molecular configuration">configuration at tetrahedral centers</a>, and double bond geometry.  These are structural features that cannot be specified by connectivity alone and SMILES which encode this information are termed isomeric SMILES.  A notable feature of these rules is that they allow rigorous partial specification of chirality.  The term isomeric SMILES is also applied to SMILES in which <a href="/wiki/Isotope" title="Isotope">isotopes</a> are specified.
</p>
<h2><span class="mw-headline" id="Graph-based_definition">Graph-based definition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=3" title="Edit section: Graph-based definition">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In terms of a graph-based computational procedure, SMILES is a string obtained by printing the symbol nodes encountered in a <a href="/wiki/Depth-first_search" title="Depth-first search">depth-first</a> <a href="/wiki/Tree_traversal" title="Tree traversal">tree traversal</a> of a <a class="mw-redirect" href="/wiki/Chemical_graph" title="Chemical graph">chemical graph</a>. The chemical graph is first trimmed to remove hydrogen atoms and cycles are broken to turn it into a <a class="mw-redirect" href="/wiki/Spanning_tree_(mathematics)" title="Spanning tree (mathematics)">spanning tree</a>. Where cycles have been broken, numeric suffix labels are included to indicate the connected nodes. Parentheses are used to indicate points of branching on the tree.
</p><p>The resultant SMILES form depends on the choices:
</p>
<ul><li>of the bonds chosen to break cycles,</li>
<li>of the starting atom used for the depth-first traversal, and</li>
<li>of the order in which branches are listed when encountered.</li></ul>
<h2><span class="mw-headline" id="Description">Description</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=4" title="Edit section: Description">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Atoms">Atoms</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=5" title="Edit section: Atoms">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Atom" title="Atom">Atoms</a> are represented by the standard abbreviation of the <a href="/wiki/Chemical_element" title="Chemical element">chemical elements</a>, in square brackets, such as [Au] for <a href="/wiki/Gold" title="Gold">gold</a>. Brackets may be omitted in the common case of atoms which:
</p>
<ol><li>are in the "organic subset" of B, C, N, O, P, S, F, Cl, Br, or I, and</li>
<li>have no formal charge, and</li>
<li>have the number of hydrogens attached implied by the SMILES valence model (typically their normal valence, but for N and P it is 3 or 5, and for S it is 2, 4 or 6), and</li>
<li>are the normal <a href="/wiki/Isotope" title="Isotope">isotopes</a>, and</li>
<li>are not chiral centers.</li></ol>
<p>All other elements must be enclosed in brackets, and have charges and hydrogens shown explicitly.  For instance, the SMILES for <a href="/wiki/Water" title="Water">water</a> may be written as either O or [OH2].  Hydrogen may also be written as a separate atom; water may also be written as [H]O[H].
</p><p>When brackets are used, the symbol H is added if the atom in brackets is bonded to one or more hydrogen, followed by the number of hydrogen atoms if greater than 1, then by the sign '+' for a positive charge or by '-' for a negative charge.   For example, [NH4+] for <a href="/wiki/Ammonium" title="Ammonium">ammonium</a>.  If there is more than one charge, it is normally written as digit; however, it is also possible to repeat the sign as many times as the ion has charges: one may write either [Ti+4] or [Ti++++] for <a href="/wiki/Titanium" title="Titanium">Titanium</a> IV (Ti<sup>4+</sup>). Thus, the <a href="/wiki/Hydroxide" title="Hydroxide">hydroxide</a> <a class="mw-redirect" href="/wiki/Anion" title="Anion">anion</a> is represented by [OH-], the <a href="/wiki/Hydronium" title="Hydronium">hydronium</a> cation is [OH3+] and the <a href="/wiki/Cobalt" title="Cobalt">cobalt</a> III <a class="mw-redirect" href="/wiki/Cation" title="Cation">cation</a> (Co<sup>3+</sup>) is either [Co+3] or [Co+++].
</p>
<h3><span class="mw-headline" id="Bonds">Bonds</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=6" title="Edit section: Bonds">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A bond is represented using one of the symbols '.' '-' '=' '#' '$' ':' '/' or '\'.
</p><p>Bonds between <a href="/wiki/Aliphatic_compound" title="Aliphatic compound">aliphatic</a> atoms are assumed to be single unless specified otherwise and are implied by adjacency in the SMILES string. Although single bonds may be written as "-", this is usually omitted.  For example, the SMILES for <a href="/wiki/Ethanol" title="Ethanol">ethanol</a> may be written as C-C-O, CC-O or C-CO, but is usually written CCO.
</p><p>Double, triple, and quadruple <a href="/wiki/Chemical_bond" title="Chemical bond">bonds</a> are represented by the symbols '=', '#', and '$' respectively as illustrated by the SMILES O=C=O (<a href="/wiki/Carbon_dioxide" title="Carbon dioxide">carbon dioxide</a>), C#N (<a href="/wiki/Hydrogen_cyanide" title="Hydrogen cyanide">hydrogen cyanide</a>) and [Ga-]$[As+] (<a href="/wiki/Gallium_arsenide" title="Gallium arsenide">gallium arsenide</a>).
</p><p>An additional type of bond is a "non-bond", indicated with ".", to indicate that two parts are not bonded together.  For example, aqueous <a href="/wiki/Sodium_chloride" title="Sodium chloride">sodium chloride</a> may be written as [Na+].[Cl-] to show the dissociation.
</p><p>An aromatic "one and a half" bond may be indicated with ':'; see <a href="#Aromaticity">§ Aromaticity</a> below.
</p><p>Single bonds adjacent to double bonds may be represented using '/' or '\' to indicate stereochemical configuration; see <a href="#Stereochemistry">§ Stereochemistry</a> below.
</p>
<h3><span class="mw-headline" id="Rings">Rings</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=7" title="Edit section: Rings">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Ring structures are written by breaking each ring at an arbitrary point (although some choices will lead to a more legible SMILES than others) to make an <a href="/wiki/Open-chain_compound" title="Open-chain compound">acyclic</a> structure and adding numerical ring closure labels to show connectivity between non-adjacent atoms.
</p><p>For example, <a href="/wiki/Cyclohexane" title="Cyclohexane">cyclohexane</a> and <a class="mw-redirect" href="/wiki/Dioxane" title="Dioxane">dioxane</a> may be written as C1CCCCC1 and O1CCOCC1 respectively.  For a second ring, the label will be 2.  For example, <a href="/wiki/Decalin" title="Decalin">decalin</a> (decahydronaphthalene) may be written as C1CCCC2C1CCCC2.
</p><p>SMILES does not require that ring numbers be used in any particular order, and permits ring number zero, although this is rarely used.  Also, it is permitted to re-use ring numbers after the first ring has closed, although this usually makes formulae harder to read.  For example, <a href="/wiki/Bicyclohexyl" title="Bicyclohexyl">bicyclohexyl</a> is usually written as C1CCCCC1C2CCCCC2, but it may also be written as C0CCCCC0C0CCCCC0.
</p><p>Multiple digits after a single atom indicate multiple ring-closing bonds.  For example, an alternative SMILES notation for decalin is C1CCCC2CCCCC12, where the final carbon participates in both ring-closing bonds 1 and 2.  If two-digit ring numbers are required, the label is preceded by %, so "C%12" is a single ring-closing bond, of ring 12.
</p><p>Ring-closing digits may be preceded by a bond type.  For example, <a href="/wiki/Cyclopropene" title="Cyclopropene">cyclopropene</a> is usually written C1=CC1, but if the double bond is chosen as the ring-closing bond, it may be written as C=1CC1, C1CC=1, or C=1CC=1.  (The first form is preferred.)  C=1CC-1 is illegal, as it explicitly specifies conflicting types for the ring-closing bond.
</p><p>Ring-closing bonds may not be used to denote multiple bonds.  For example, C1C1 is not a valid alternative to C=C for <a href="/wiki/Ethylene" title="Ethylene">ethylene</a>.  However, they may be used with non-bonds; C1.C2.C12 is a peculiar but legal alternative way to write <a href="/wiki/Propane" title="Propane">propane</a>, more commonly written CCC.
</p><p>Choosing a ring-break point adjacent to attached groups can lead to a simpler SMILES form by avoiding branches.  For example, <a href="/wiki/Cyclohexane-1,2-diol" title="Cyclohexane-1,2-diol">cyclohexane-1,2-diol</a> is most simply written as OC1CCCCC1O; choosing a different ring-break location produces a branched structure that requires parentheses to write.
</p>
<h3><span class="mw-headline" id="Aromaticity">Aromaticity</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=8" title="Edit section: Aromaticity">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Aromaticity" title="Aromaticity">Aromatic</a> rings such as <a href="/wiki/Benzene" title="Benzene">benzene</a> may be written in one of three forms:
</p>
<ol><li>In Kekulé form with alternating single and double bonds, e.g. C1=CC=CC=C1,</li>
<li>Using the aromatic bond symbol ":", e.g. C:1:C:C:C:C:C1, or</li>
<li>Most commonly, by writing the constituent B, C, N, O, P and S atoms in lower-case forms 'b', 'c', 'n', 'o', 'p' and 's', respectively.</li></ol>
<p>In the latter case, bonds between two aromatic atoms are assumed (if not explicitly shown) to be aromatic bonds.  Thus, <a href="/wiki/Benzene" title="Benzene">benzene</a>, <a href="/wiki/Pyridine" title="Pyridine">pyridine</a> and <a href="/wiki/Furan" title="Furan">furan</a> can be represented respectively by the SMILES c1ccccc1, n1ccccc1 and o1cccc1.
</p><p>Aromatic nitrogen bonded to hydrogen, as found in <a href="/wiki/Pyrrole" title="Pyrrole">pyrrole</a> must be represented as [nH] and <a href="/wiki/Imidazole" title="Imidazole">imidazole</a> is written in SMILES notation as n1c[nH]cc1.
</p><p>When aromatic atoms are singly bonded to each other, such as in <a href="/wiki/Biphenyl" title="Biphenyl">biphenyl</a>, a single bond must be shown explicitly: c1ccccc1-c2ccccc2.  This is one of the few cases where the single bond symbol "-" is required.  (In fact, most SMILES software can correctly infer that the bond between the two rings cannot be aromatic and so will accept the form "c1ccccc1c2ccccc2".)
</p><p>The Daylight and OpenEye algorithms for generating canonical SMILES differ in their treatment of aromaticity.
</p>
<div class="thumb tright"><div class="thumbinner" style="width:352px;"><a class="image" href="/wiki/File:3-cyanoanisole_SMILES.svg"><img alt="" class="thumbimage" data-file-height="171" data-file-width="625" height="96" src="//upload.wikimedia.org/wikipedia/commons/thumb/3/31/3-cyanoanisole_SMILES.svg/350px-3-cyanoanisole_SMILES.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/3/31/3-cyanoanisole_SMILES.svg/525px-3-cyanoanisole_SMILES.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/3/31/3-cyanoanisole_SMILES.svg/700px-3-cyanoanisole_SMILES.svg.png 2x" width="350"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:3-cyanoanisole_SMILES.svg" title="Enlarge"></a></div>Visualization of 3-cyanoanisole as COc(c1)cccc1C#N.</div></div></div>
<h3><span class="mw-headline" id="Branching">Branching</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=9" title="Edit section: Branching">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Branches are described with parentheses, as in CCC(=O)O for <a href="/wiki/Propionic_acid" title="Propionic acid">propionic acid</a> and FC(F)F for <a href="/wiki/Fluoroform" title="Fluoroform">fluoroform</a>.  The first atom within the parentheses, and the first atom after the parenthesized group, are both bonded to the same branch point atom.
</p><p>Substituted rings can be written with the branching point in the ring as illustrated by the SMILES COc(c1)cccc1C#N (<a class="external text" href="https://web.archive.org/web/20130522091354/http://www.daylight.com/daycgi/depict?434f6328633129636363633143234e" rel="nofollow">see depiction</a>) and COc(cc1)ccc1C#N (<a class="external text" href="https://web.archive.org/web/20130522074308/http://www.daylight.com/daycgi/depict?434f6328636331296363633143234e" rel="nofollow">see depiction</a>) which encode the 3 and 4-cyanoanisole isomers.  Writing SMILES for substituted rings in this way can make them more human-readable.
</p><p>Branches may be written in any order.  For example, <a href="/wiki/Bromochlorodifluoromethane" title="Bromochlorodifluoromethane">bromochlorodifluoromethane</a> may be written as FC(Br)(Cl)F, BrC(F)(F)Cl, C(F)(Cl)(F)Br, or the like.  Generally, a SMILES form is easiest to read if the simpler branch comes first, with the final, unparenthesized portion being the most complex.  The only caveats to such rearrangements are:
</p>
<ul><li>If ring numbers are reused, they are paired according to their order of appearance in the SMILES string.  Some adjustments may be required to preserve the correct pairing.</li>
<li>If stereochemistry is specified, adjustments must be made; see <a href="/wiki/Stereochemistry#Notes" title="Stereochemistry">Stereochemistry § Notes</a> below.</li></ul>
<p>The one form of branch which does <i>not</i> require parentheses are ring-closing bonds.  Choosing ring-closing bonds appropriately can reduce the number of parentheses required.  For example, <a href="/wiki/Toluene" title="Toluene">toluene</a> is normally written as Cc1ccccc1 or c1ccccc1C, avoiding the parentheses required if written as c1ccc(C)ccc1 or c1ccc(ccc1)C.
</p>
<h3><span class="mw-headline" id="Stereochemistry">Stereochemistry</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=10" title="Edit section: Stereochemistry">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright"><div class="thumbinner" style="width:112px;"><a class="image" href="/wiki/File:Trans-1,2-difluoroethylene.svg"><img alt="" class="thumbimage" data-file-height="180" data-file-width="180" height="110" src="//upload.wikimedia.org/wikipedia/commons/thumb/c/c3/Trans-1%2C2-difluoroethylene.svg/110px-Trans-1%2C2-difluoroethylene.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/c/c3/Trans-1%2C2-difluoroethylene.svg/165px-Trans-1%2C2-difluoroethylene.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/c/c3/Trans-1%2C2-difluoroethylene.svg/220px-Trans-1%2C2-difluoroethylene.svg.png 2x" width="110"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:Trans-1,2-difluoroethylene.svg" title="Enlarge"></a></div><i>trans</i>-1,2-difluoroethylene</div></div></div>
<p>SMILES permits, but does not require, specification of <a class="mw-redirect" href="/wiki/Stereoisomer" title="Stereoisomer">stereoisomers</a>.
</p><p>Configuration around double bonds is specified using the characters "/" and "\" to show directional single bonds adjacent to a double bond. For example, F/C=C/F (<a class="external text" href="https://web.archive.org/web/20130522072357/http://www.daylight.com/daycgi/depict?462f433d432f46" rel="nofollow">see depiction</a>) is one representation of <i><a class="mw-redirect" href="/wiki/Trans_isomer" title="Trans isomer">trans</a></i>-<a class="mw-redirect" href="/wiki/1,2-difluoroethylene" title="1,2-difluoroethylene">1,2-difluoroethylene</a>, in which the fluorine atoms are on opposite sides of the double bond (as shown in the figure), whereas F/C=C\F (<a class="external text" href="https://web.archive.org/web/20130522074206/http://www.daylight.com/daycgi/depict?462f433d435c46" rel="nofollow">see depiction</a>) is one possible representation of <i><a class="mw-redirect" href="/wiki/Cis-trans_isomerism" title="Cis-trans isomerism">cis</a></i>-1,2-difluoroethylene, in which the Fs are on the same side of the double bond.
</p><p>Bond direction symbols always come in groups of at least two, of which the first is arbitrary.  That is, F\C=C\F is the same as F/C=C/F.  When alternating single-double bonds are present, the groups are larger than two, with the middle directional symbols being adjacent to two double bonds.  For example, the common form of (2,4)-hexadiene is written C/C=C/C=C/C.
</p>
<div class="thumb tright"><div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Beta-Carotene_conjugation.svg"><img alt="" class="thumbimage" data-file-height="91" data-file-width="443" height="45" src="//upload.wikimedia.org/wikipedia/commons/thumb/8/83/Beta-Carotene_conjugation.svg/220px-Beta-Carotene_conjugation.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/8/83/Beta-Carotene_conjugation.svg/330px-Beta-Carotene_conjugation.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/8/83/Beta-Carotene_conjugation.svg/440px-Beta-Carotene_conjugation.svg.png 2x" width="220"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:Beta-Carotene_conjugation.svg" title="Enlarge"></a></div><a class="mw-redirect" href="/wiki/Beta-carotene" title="Beta-carotene">Beta-carotene</a>, with the eleven double bonds highlighted.</div></div></div>
<p>As a more complex example, <a class="mw-redirect" href="/wiki/Beta-carotene" title="Beta-carotene">beta-carotene</a> has a very long backbone of alternating single and double bonds, which may be written CC1CCC/C(C)=C1/C=C/C(C)=C/C=C/C(C)=C/C=C/C=C(C)/C=C/C=C(C)/C=C/C2=C(C)/CCCC2(C)C.
</p><p>Configuration at <a href="/wiki/Stereocenter" title="Stereocenter">tetrahedral carbon</a> is specified by @ or @@.  Consider the four bonds in the order in which they appear, left to right, in the SMILES form.  Looking toward the central carbon from the perspective of the first bond, the other three are either clockwise or counter-clockwise.  These cases are indicated with @@ and @, respectively.  (Because the @ symbol itself is a counter-clockwise spiral.)
</p>
<div class="thumb tright"><div class="thumbinner" style="width:112px;"><a class="image" href="/wiki/File:L-Alanin_-_L-Alanine.svg"><img alt="" class="thumbimage" data-file-height="121" data-file-width="163" height="82" src="//upload.wikimedia.org/wikipedia/commons/thumb/9/90/L-Alanin_-_L-Alanine.svg/110px-L-Alanin_-_L-Alanine.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/9/90/L-Alanin_-_L-Alanine.svg/165px-L-Alanin_-_L-Alanine.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/9/90/L-Alanin_-_L-Alanine.svg/220px-L-Alanin_-_L-Alanine.svg.png 2x" width="110"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:L-Alanin_-_L-Alanine.svg" title="Enlarge"></a></div>L-alanine</div></div></div>
<p>For example, consider the <a href="/wiki/Amino_acid" title="Amino acid">amino acid</a> <a href="/wiki/Alanine" title="Alanine">alanine</a>.  One of its SMILES forms is NC(C)C(=O)O, more fully written as N[CH](C)C(=O)O.  L-alanine, the more common <a href="/wiki/Enantiomer" title="Enantiomer">enantiomer</a>, is written as N[C@@H](C)C(=O)O (<a class="external text" href="https://web.archive.org/web/20130704043108/http://www.daylight.com/daycgi/depict?4e5b434040485d28432943283d4f294f" rel="nofollow">see depiction</a>).  Looking from the N-C bond, the hydrogen (H), methyl (C), and carboxylate (C(=O)O) groups appear clockwise.  D-Alanine can be written as N[C@H](C)C(=O)O (<a class="external text" href="https://web.archive.org/web/20130522072012/http://www.daylight.com/daycgi/depict?4e5b4340485d28432943283d4f294f" rel="nofollow">see depiction</a>).
</p><p>While the order is which branches are specified in SMILES is normally unimportant, in this case it matters; swapping any two groups requires reversing the chirality indicator.  If the branches are reversed so alanine is written as NC(C(=O)O)C, then the configuration also reverses; L-alanine is written as N[C@H](C(=O)O)C (<a class="external text" href="https://web.archive.org/web/20130522073747/http://www.daylight.com/daycgi/depict?4e5b434040485d2843283d4f294f2943" rel="nofollow">see depiction</a>).  Other ways of writing it include C[C@H](N)C(=O)O, OC(=O)[C@@H](N)C and OC(=O)[C@H](C)N.
</p><p>Normally, the first of the four bonds appears to the left of the carbon atom, but if the SMILES is written beginning with the chiral carbon, such as C(C)(N)C(=O)O, then all four are to the right, but the first to appear (the [CH] bond in this case) is used as the reference to order the following three: L-alanine may also be written [C@@H](C)(N)C(=O)O.
</p><p>The SMILES specification includes elaborations on the @ symbol to indicate stereochemistry around more complex chiral centers, such as <a href="/wiki/Trigonal_bipyramidal_molecular_geometry" title="Trigonal bipyramidal molecular geometry">trigonal bipyramidal molecular geometry</a>.
</p>
<h3><span class="mw-headline" id="Isotopes">Isotopes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=11" title="Edit section: Isotopes">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a class="mw-redirect" href="/wiki/Isotopes" title="Isotopes">Isotopes</a> are specified with a number equal to the integer isotopic mass preceding the atomic symbol.  <a href="/wiki/Benzene" title="Benzene">Benzene</a> in which one atom is <a href="/wiki/Carbon-14" title="Carbon-14">carbon-14</a> is written as [14c]1ccccc1 and <a class="mw-redirect" href="/wiki/Deuterochloroform" title="Deuterochloroform">deuterochloroform</a> is [2H]C(Cl)(Cl)Cl.
</p>

<h3><span class="mw-headline" id="Other_examples_of_SMILES">Other examples of SMILES</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=13" title="Edit section: Other examples of SMILES">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The SMILES notation is described extensively in the SMILES theory manual provided by <a class="new" href="/w/index.php?title=Daylight_Chemical_Information_Systems&amp;action=edit&amp;redlink=1" title="Daylight Chemical Information Systems (page does not exist)">Daylight Chemical Information Systems</a> and a number of illustrative examples are presented. Daylight's depict utility provides users with the means to check their own examples of SMILES and is a valuable educational tool.
</p>
<h2><span class="mw-headline" id="Extensions">Extensions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=14" title="Edit section: Extensions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/Smiles_arbitrary_target_specification" title="Smiles arbitrary target specification">SMARTS</a> is a line notation for specification of substructural patterns in molecules.  While it uses many of the same symbols as SMILES, it also allows specification of <a href="/wiki/Wildcard_character" title="Wildcard character">wildcard</a> atoms and bonds, which can be used to define substructural queries for <a href="/wiki/Chemical_database" title="Chemical database">chemical database</a> searching. One common misconception is that SMARTS-based substructural searching involves matching of SMILES and SMARTS strings. In fact, both SMILES and SMARTS strings are first converted to internal graph representations which are searched for <a class="mw-redirect" href="/wiki/Glossary_of_graph_theory#Subgraphs" title="Glossary of graph theory">subgraph</a> <a href="/wiki/Isomorphism" title="Isomorphism">isomorphism</a>.
</p><p><span id="SMIRKS"></span>SMIRKS, a superset of "reaction SMILES" and a subset of "reaction SMARTS", is a line notation for specifying reaction transforms. The general syntax for the reaction extensions is <code>REACTANT&gt;AGENT&gt;PRODUCT</code> (without spaces), where any of the fields can either be left blank or filled with multiple molecules deliminated with a dot (<code>.</code>), and other descriptions dependent on the base language. Atoms can additionally be identified with a number (e.g. <code>[C:1]</code>) for mapping,<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> for example in <code><a class="external text" href="http://cdb.ics.uci.edu/cgibin/Smi2DepictWeb.py?smiles=%5BCH2%3A1%5D%3D%5BCH%3A2%5D%5BCH%3A3%5D%3D%5BCH%3A4%5D%5BCH2%3A5%5D%5BH%3A6%5D%3E%3E%5BH%3A6%5D%5BCH2%3A1%5D%5BCH%3A2%5D%3D%5BCH%3A3%5D%5BCH%3A4%5D%3D%5BCH2%3A5%5D" rel="nofollow">[CH2:1]=[CH:2][CH:3]=[CH:4][CH2:5][H:6]&gt;&gt;[H:6][CH2:1][CH:2]=[CH:3][CH:4]=[CH2:5]</a></code>.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup>
</p>
<h2><span class="mw-headline" id="Conversion">Conversion</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=15" title="Edit section: Conversion">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>SMILES can be converted back to 2-dimensional representations using Structure Diagram Generation algorithms (Helson, 1999). This conversion is not always unambiguous. Conversion to 3-dimensional representation is achieved by energy-minimization approaches. There are many downloadable and web-based conversion utilities.
</p>

<h3><span class="mw-headline" id="SMILES_related_software_utilities">SMILES related software utilities</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simplified_molecular-input_line-entry_system&amp;action=edit&amp;section=20" title="Edit section: SMILES related software utilities">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul><li><a href="/wiki/National_Institutes_of_Health" title="National Institutes of Health">NIH</a> online services
<ul><li><a class="external text" href="https://cactus.nci.nih.gov/chemical/structure" rel="nofollow">NCI/CADD Chemical Identifier Resolver</a> – resolves or generates SMILES from chemical names, CAS Registry Numbers, InChI/InChIKey and many other chemical structure file formats</li>
<li><a class="external text" href="https://cactus.nci.nih.gov/gifcreator/" rel="nofollow">GIF/PNG-Creator</a> for 2D Plots of Chemical Structures</li>
<li><a class="external text" href="https://pubchem.ncbi.nlm.nih.gov/edit/index.html" rel="nofollow">PubChem server side structure editor</a> – online molecule editor</li>
<li><a class="external text" href="https://cactus.nci.nih.gov/translate/" rel="nofollow">NCI/CADD Online SMILES Translator and Structure File Generator</a> – <a class="mw-redirect" href="/wiki/JSME_Molecule_Editor" title="JSME Molecule Editor">JSME</a> online molecule editor that generates SMILES/SMARTS; <a class="external text" href="http://peter-ertl.com/jsme/" rel="nofollow">source code</a> (BSD license).</li></ul></li>
<li><a href="/wiki/ChemAxon" title="ChemAxon">ChemAxon</a> utilties, mostly Java-based, some with free personal use
<ul><li><a class="external text" href="http://cdb.ics.uci.edu/cgibin/Smi2DepictWeb.py" rel="nofollow">Smi2DepictWeb</a> - Translate a SMILES formula into graphics with Marvin, hosted by <a class="mw-redirect" href="/wiki/UC_Irvine" title="UC Irvine">UC Irvine</a> (other online tools available on <a class="external text" href="http://cdb.ics.uci.edu/" rel="nofollow">the same site</a>)</li>
<li><a class="external text" href="https://chemaxon.com/products/marvin" rel="nofollow">Marvin</a> – chemical editor/viewer and SMILES generator/converter (<a class="external text" href="https://web.archive.org/web/20071107150216/http://www.chemaxon.com/product/live_examples.html" rel="nofollow">previous version</a>)</li>
<li><a class="external text" href="https://chemaxon.com/products/instant-jchem" rel="nofollow">Instant JChem</a> – desktop application for storing/generating/converting/visualizing/searching SMILES structures, particularly batch processing (<a class="external text" href="https://web.archive.org/web/20071112191748/http://www.chemaxon.com/product/ijc.html" rel="nofollow">previous version</a>)</li>
<li>other tools and third-party integration; <a class="external text" href="https://chemaxon.com/products" rel="nofollow">full list</a></li></ul></li>
<li><a href="/wiki/OELib" title="OELib">OELib</a> and descendents (<a class="external text" href="http://openbabel.org/" rel="nofollow">Open Babel</a> etc.; see also "<a href="#see_also">#see also</a>")
<ul><li><a class="external text" href="http://www.hungry.com/~alves/smormoed/" rel="nofollow">Smormo-Ed</a> – a FOSS molecule editor which can read and write SMILES; has <a class="mw-redirect" href="/wiki/Gtk%2B" title="Gtk+">Gtk+</a> and <a href="/wiki/HTML5" title="HTML5">HTML5</a> frontends</li></ul></li>
<li><a class="external text" href="https://web.archive.org/web/20070914100359/http://www.chembiogrid.org/cheminfo/smi23d/" rel="nofollow">smi23d</a> – 3D Coordinate Generation; <a class="external text" href="https://sourceforge.net/p/cicc-grid/code/HEAD/tree/cicc-grid/smi23d/" rel="nofollow">source code archive</a></li>
<li><a class="external text" href="http://www.acdlabs.com/download/chemsk.html" rel="nofollow">ACD/ChemSketch</a></li>
<li><a class="external text" href="http://inchi.info/" rel="nofollow">InChI.info</a> – an unofficial InChI website featuring on-line converter from InChI and SMILES to molecular drawings, based on <a href="/wiki/BKChem" title="BKChem">OASA</a></li>
<li><a class="external text" href="http://web.abo.fi/~mivainio/balloon/" rel="nofollow">Balloon</a> – A free program for 3D coordinate generation and conformational analysis.</li>
<li><a class="external text" href="http://lifescience.opensource.epam.com/indigo/index.html" rel="nofollow">Indigo</a> – an open-source cross-platform cheminformatics library with a plugin for IUPAC-compliant molecule and reaction 2D structural formula rendering.</li>
<li><a class="external text" href="http://bioclipse.net/" rel="nofollow">Bioclipse</a> – a free and open source workbench for the life sciences (<a href="/wiki/Bioclipse" title="Bioclipse">Wikipedia article</a>)</li>
<li>Scilligence utilities
<ul><li><a class="external text" href="http://www.molengine.com/" rel="nofollow">MolEngine</a> – A .NET cheminformatics toolkit to read/write SMILES, generate 2D coordinate from SMILES, and convert SMILES from/into other <a href="/wiki/Chemical_file_format" title="Chemical file format">Chemical file formats</a>.</li>
<li><a class="external text" href="http://www.jsdraw.com/" rel="nofollow">JSDraw</a> – A cross-platform javascript chemical structure editor to generate SMILES and SMARTS.</li></ul></li></ul>
